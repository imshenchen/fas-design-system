# 功能 Spec：新增節點
**系統**：SIE - Smart Integration Edgebox（智能整合邊緣運算機控制台）  
**路徑**：多節點管理 > 新增節點  
**版本**：EAP+ 1.4.0  
**整理日期**：2026-03-31

---

## 一、功能概述

管理者可透過兩種方式將邊緣節點加入多節點管理清單：

| 方式 | 說明 |
|------|------|
| **手動添加節點** | 逐筆輸入或透過 TSV 批次匯入全新節點 |
| **添加自候選節點清單** | 從「曾被移除的節點」清單中重新選取並加入 |

---

## 二、入口路徑

```
多節點管理（列表頁）
  └─ 右上角「新增節點」按鈕（下拉選單）
        ├─ 添加自候選節點清單
        └─ 手動添加節點
```

---

## 三、候選節點的資料來源說明

> **重要邏輯**：候選節點清單的內容，來自於「曾經透過手動添加、但後來從節點管理清單中被移除」的節點。移除後節點不會被刪除，而是退回候選清單，可被重新加入。

- 候選節點本身有自己的**名稱**（系統層的識別名稱）
- 節點加入管理清單後會有**別名（Alias）**，兩者可以不同
- 從候選節點清單加入時，系統會要求使用者重新輸入別名

## 四、功能一：添加自候選節點清單

### 4-1 頁面結構

```
多節點管理 > 添加自候選節點清單
├── 頂部工具列
│   ├── Breadcrumb：多節點管理 > 添加自候選節點清單
│   └── 右上角「手動添加節點」快速入口按鈕
├── 搜尋與篩選列
│   ├── 關鍵字搜尋（節點名稱、地址、連接埠）
│   ├── 狀態篩選（所有狀態）
│   └── 使用狀態篩選（所有使用狀態）
└── 候選節點列表（表格）
    ├── 分頁控制（每頁列數：10，預設）
    └── 各欄位（見下方）
```

### 4-2 列表欄位

| 欄位 | 說明 |
|------|------|
| 名稱 | 候選節點本身的名稱（非別名），可排序 |
| 地址 | 節點的 IP 位址，可排序 |
| 連接埠 | 節點的連接埠號，可排序 |
| 狀態 | 節點連線狀態，顯示為徽章：`確定`（綠）/ `危急`（紅），可排序 |
| 健康訊息 | 系統健康檢查的 HTTP 端點 URL（截斷顯示），可排序 |
| 註冊 ID | 節點的唯一識別碼（截斷顯示），可排序 |
| 使用狀態 | `使用中`（藍點）/ `閒置`（灰點），可排序 |
| 設定 | 操作按鈕：複製 📋、刪除 🗑️ |

### 4-3 加入節點 Dialog 欄位

點擊列表列的複製圖示（📋）後，彈出「向主節點添加節點」Dialog：

**唯讀資訊區（來自候選節點，不可編輯）**

| 欄位 | 說明 |
|------|------|
| 候選節點名稱 | 該候選節點本身的名稱 |
| 地址 | 節點的 IP 位址 |
| 連接埠 | 節點的連接埠號 |

**SIE 節點資訊區（使用者填寫）**

| 欄位名稱 | 必填 | 類型 | 限制 |
|----------|------|------|------|
| *別名 | ✅ 必填 | 文字輸入 | 最多 30 字元（顯示 0/30 計數） |
| 標籤 | ❌ 選填 | Searchable Dropdown | 可搜尋既有標籤 |
| 說明 | ❌ 選填 | 多行文字輸入 | 最多 100 字元（顯示 0/100 計數） |

**Dialog 按鈕**

| 按鈕 | 狀態 | 說明 |
|------|------|------|
| 取消 | 常態啟用 | 關閉 Dialog，不做任何變更 |
| 加入 | 別名未填時 disabled | 送出並將節點加入管理清單 |

## 五、功能二：手動添加節點

### 5-1 頁面結構

```
多節點管理 > 手動添加節點
├── 頂部工具列
│   ├── Breadcrumb：多節點管理 > 手動添加節點
│   └── 右上角「送出」按鈕（有已確認列時啟用，否則 disabled）
├── 區塊 A：TSV 批次匯入
│   ├── 下載模板連結（下載模板.tsv）
│   ├── 檔案上傳區（拖曳 or 點選「選擇檔案」）
│   └── 說明文字：請上傳節點列表，或使用下面的編輯器
└── 區塊 B：手動輸入列表（表格）
    └── 資料行（支援多行，逐筆加入）
```

### 5-2 TSV 批次匯入

| 元件 | 說明 |
|------|------|
| 下載模板.tsv | 提供標準欄位格式的 TSV 範本 |
| 選擇檔案（上傳區） | 支援格式：TSV；最大檔案尺寸：1MB |

### 5-3 手動輸入表格欄位

| # | 欄位名稱 | 必填 | 類型 | 說明 |
|---|----------|------|------|------|
| 1 | *別名 | ✅ 必填 | 文字輸入 | 節點在管理清單中的顯示名稱 |
| 2 | 說明 | ❌ 選填 | 文字輸入 | 節點的備註或描述 |
| 3 | 標籤 | ❌ 選填 | Searchable Dropdown | 可搜尋並選取既有標籤 |
| 4 | *地址 | ✅ 必填 | 文字輸入 | 節點的 IP 位址 |
| 5 | *連接埠 | ✅ 必填 | 數字輸入 | 節點的連接埠號 |

### 5-4 列操作按鈕

| 按鈕 | 出現時機 | 說明 |
|------|----------|------|
| 加入 | 該列尚未確認（填寫中） | 確認該行資料，新增下一筆空白列 |
| 刪除（🗑️） | 該列已被確認後 | 刪除該筆已確認的資料列 |

## 六、Use Cases

### UC-01：從候選節點清單加入節點（Happy Path）

**角色**：管理員（sie_admin）  
**前提**：候選節點清單中有至少一筆節點

| 步驟 | 使用者動作 | 系統回應 |
|------|-----------|---------|
| 1 | 點擊「新增節點」> 選擇「添加自候選節點清單」 | 跳轉至候選節點清單頁面 |
| 2 | 瀏覽或搜尋候選節點 | 列表即時過濾顯示 |
| 3 | 點擊目標節點右側的複製圖示（📋） | 彈出「向主節點添加節點」Dialog，顯示候選節點資訊 |
| 4 | 填寫別名（必填），選填標籤與說明 | 別名輸入後「加入」按鈕啟用 |
| 5 | 點擊「加入」 | Dialog 關閉，返回多節點管理列表，顯示「節點已成功添加。」Toast 通知，節點出現於清單 |

---

### UC-02：手動逐筆新增節點（Happy Path）

**前提**：位於多節點管理頁面

| 步驟 | 使用者動作 | 系統回應 |
|------|-----------|---------|
| 1 | 點擊「新增節點」> 選擇「手動添加節點」 | 跳轉至手動添加頁面，顯示空白第 1 列 |
| 2 | 填寫節點資料（別名、地址、連接埠為必填） | 即時顯示輸入內容 |
| 3 | 點擊「加入」 | 第 1 列確認，顯示刪除按鈕；自動新增第 2 列空白行 |
| 4 | 重複步驟 2–3（可選） | 每次加入後列序號遞增 |
| 5 | 點擊右上角「送出」 | 提交所有節點，返回管理清單，顯示成功 Toast |

---

### UC-03：透過 TSV 批次匯入節點

| 步驟 | 使用者動作 | 系統回應 |
|------|-----------|---------|
| 1 | 點擊「下載模板.tsv」 | 下載 TSV 模板，Toast 提示「準備開始下載」 |
| 2 | 填妥模板後，點擊「選擇檔案」上傳 | 系統解析 TSV，將節點資料填入下方表格 |
| 3 | 確認資料後點擊「送出」 | 批次提交，返回管理清單 |

---

### UC-04：刪除已加入的手動輸入列

| 步驟 | 使用者動作 | 系統回應 |
|------|-----------|---------|
| 1 | 點擊已確認列右側的 🗑️ | 該列從表格移除，後續列重新排序 |

---

### UC-05：從候選節點清單加入頁面切換至手動添加

| 步驟 | 使用者動作 | 系統回應 |
|------|-----------|---------|
| 1 | 在候選節點清單頁面，點擊右上角「手動添加節點」按鈕 | 直接跳轉至手動添加節點頁面 |

## 七、例外與 Edge Cases

### 7-1 表單驗證

| 情境 | 預期行為 |
|------|---------|
| 手動輸入列：必填欄位（別名 / 地址 / 連接埠）未填，點擊「加入」 | 觸發欄位驗證錯誤提示，無法確認該列 |
| 手動輸入列：連接埠輸入非數字 | 觸發格式錯誤提示 |
| 手動輸入列：連接埠超出有效範圍（1–65535） | 觸發範圍錯誤提示 |
| 手動輸入列：地址格式非合法 IP 或 hostname | 觸發格式錯誤提示 |
| 手動輸入列：別名超過字元上限 | 觸發長度錯誤提示 |
| Dialog：別名未填，點擊「加入」 | 「加入」按鈕維持 disabled，無法送出 |
| Dialog：別名超過 30 字元 | 觸發長度錯誤提示（計數器顯示 30/30） |
| Dialog：說明超過 100 字元 | 觸發長度錯誤提示（計數器顯示 100/100） |

### 7-2 重複節點

| 情境 | 預期行為 |
|------|---------|
| 手動添加：地址 + 連接埠組合已存在於管理清單 | 系統應提示重複節點錯誤（送出時或加入時攔截） |
| 候選節點加入：該節點已在管理清單中（使用狀態為「使用中」） | 預期攔截或警示，避免重複加入 |
| 手動添加：別名與現有節點重複 | 是否允許重名待確認（系統未明確限制） |

### 7-3 TSV 匯入

| 情境 | 預期行為 |
|------|---------|
| 上傳非 TSV 格式（如 .csv、.xlsx） | 顯示格式錯誤提示，不解析 |
| 檔案超過 1MB | 顯示尺寸超限錯誤提示，拒絕上傳 |
| TSV 內容格式錯誤（欄位缺失、順序錯誤） | 顯示解析錯誤提示 |
| TSV 內部分列不合法、部分合法 | 待確認：是否允許部分列匯入，或全部拒絕 |
| TSV 檔案為空 | 顯示空檔案錯誤提示 |

### 7-4 送出 / 系統回應

| 情境 | 預期行為 |
|------|---------|
| 手動添加頁：無任何已確認列，嘗試送出 | 「送出」按鈕維持 disabled，無法點擊 |
| 有未確認列（填了資料但未按「加入」）直接按送出 | 待確認：是否自動加入該列，或忽略，或提示 |
| 送出後 API 失敗（網路錯誤 / 伺服器錯誤） | 顯示錯誤 Toast 或 Dialog，節點不被加入 |
| 批次送出時部分成功、部分失敗 | 待確認：系統需說明哪幾筆失敗及原因 |
| 節點加入成功後連線失敗（節點不可達） | 節點出現於清單，狀態顯示為「危急」（紅色徽章） |

### 7-5 操作行為

| 情境 | 預期行為 |
|------|---------|
| 填寫到一半離開頁面（按 Breadcrumb 或側邊選單） | 待確認：是否顯示「確定要離開？」提示 |
| 同時上傳 TSV 又手動填寫列 | 兩者資料合併於表格中，一起送出 |
| 候選節點清單為空（無任何候選節點） | 待確認：顯示空狀態提示，如「目前無候選節點」 |

## 十、補充觀察與已知問題

- 節點名稱顯示「[object Object]」（Image 3）：候選節點加入後別名異常渲染，疑為 Bug，候選節點的名稱欄位傳入了物件而非字串
- 「手動添加節點」頁面右上角有從「候選節點清單」頁面跳轉過來的快速按鈕，反向亦然（候選節點清單右上角有「手動添加節點」按鈕），兩個入口互相連通
- 移除節點時，系統會提示「該節點從節點管理列表中移除後，仍將存在於候選節點列表中。」確保資料不遺失
